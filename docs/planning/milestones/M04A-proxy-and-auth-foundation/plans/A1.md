## 0. Scope decisions (read first)

A1 delivers four things:

1. **An outbound, transport-agnostic service proxy** (`ServiceProxy` trait in
   `syneroym-rpc`; `ProxyRouter` impl in `syneroym-router`) that routes a typed
   `(service, interface, method, params)` call to
   (a) a local native service, (b) a local WASM component, or (c) a **remote
   node over Iroh QUIC + JSON-RPC**, with retry/backoff hook points.
2. **A guest-facing WIT interface** (`syneroym:proxy@0.1.0`) so a WASM component
   can originate such a call — today a guest has *no* way to call another
   service, so "component↔component calls" is unreachable without it.
3. **Fully typed WASM results** (`Value`, not the string-or-debug boundary
   contract) — A0′ explicitly deferred this to A1
   (`conversions.rs::wasm_results_to_json_string` doc comment: "Fully typing the
   `result` field is Slice A1").
4. **Typed unsupported-protocol/version errors** on the inbound path for a
   scheme this node does not speak (`wrpc://`, any `Other(_)`).

Non-goals, restated: no wRPC wire (A.5), no protocol negotiation (A.7), no DLQ/
outbox (M5), no dynamic WIT import interception / late binding (the
`system-architecture.md:1930` "inject dynamically generated proxy host
functions" vision — see §Flag 6).

## 1. Flags: things in `task.md` that are stale, ambiguous, or wrong

Resolve these before/while implementing; each has a recommendation, none is
guessed silently.

**F1 — "replacing the `dispatch.rs:122-123` stub" is a mis-anchor.**
The stub is the `(AdaptationStage::JsonRpcToWrpc, ServiceStage::WasmComponent)`
arm (now `dispatch.rs:138-142`). `plan_pipeline` **never produces
`AdaptationStage::JsonRpcToWrpc`** — grep it: no match arm yields that variant.
The arm is unreachable dead code, and it is an *inbound* concern, whereas the
Universal Proxy is an *outbound* one. **Recommendation:** keep the
`AdaptationStage::JsonRpcToWrpc` variant reserved (A.5 says the seam stays), do
**not** build the proxy "behind" it, and instead satisfy the task's real intent
— a transport-agnostic seam a future wRPC wire slots into additively — with
`ProxyProtocol` + the `RemoteHop` trait (§5). The stub arm's message becomes the
typed unsupported-protocol error (§7).

**F2 — `(RouteProtocol::Wrpc, WasmChannel)` currently lands in the ADR-0014
stream path.** `plan_pipeline` maps it to `ServiceStage::WasmComponent` and then
force-overrides transport to `Raw`; `handle_raw_stream`'s `WasmComponent` arm is
now the Slice 6B stream-protocol handler. So a `wrpc://` request today fails with
"missing or invalid `dir` query parameter" — a confusing, wrong error for an
unimplemented protocol. §7 fixes this.

**F3 — the Failure-tests row "Peer presents a delegation cert for a *different*
service's DID → Rejected by `verify_preamble` (existing check, unchanged)" is
not true of the current code.** `HandshakeVerifier::verify_preamble`
(`handshake.rs:43-92`) checks `cert.verify(master_did)`, `cert.temporary_did ==
did(preamble.pubkey)`, and DHT revocation. It **never** compares the cert against
`preamble.service_id`. The cert binds *master → temporary key*, not *caller →
callee*. This matters for A1 because proof-forwarding (§5.4) relies on exactly
this property (a proof is portable across callees). **Recommendation:** correct
the row in `task.md` to "cert whose `temporary_did` does not match the preamble
pubkey → rejected"; do not add a callee-binding check in A1 (it would be a new
authorization model, and B1/UCAN is where audience-binding belongs).

**F4 — Goal says "WASM↔Podman/TCP" is on the JSON-RPC uniform wire; Slice A1's
body says only "component↔component and native↔native".** There is no framing/
JSON-RPC contract for a `SubstrateEndpoint::TcpHostPort` service anywhere in the
repo (TCP endpoints are byte-passthrough proxies today). **Recommendation:**
A1 returns a typed `unsupported-target` error (`-32093`) for `TcpHostPort`
targets, and `task.md`'s Goal is corrected to say TCP/Podman JSON-RPC targets are
deferred. Adding it later is one match arm.

**F5 — "WIT Boundary Versioning" in Migration Strategy lists `syneroym:iam` and
`data-layer` additions, but no proxy package** — yet a guest-originated proxy
call is impossible without one. **Recommendation:** add
`syneroym:proxy@0.1.0` (§3) and record it in `task.md`'s Migration Strategy.

**F6 — the architecture doc's Universal Proxy is "WIT import interception + late
binding" (generated proxy host functions satisfying `import acme:booking/service`).**
That requires synthesizing wasmtime linker entries for arbitrary, unknown WIT
worlds at deploy time. A1 delivers the *routing/identity/retry* substance of the
proxy with an **explicit** call interface instead. **Recommendation:** state this
explicitly in `status.md` and add a note at `system-architecture.md:1930`; late
binding is a separate (M5+) slice, not silently "done".

**F7 — `RouteHandlerInner.iroh_endpoint` is `None` on every real substrate node**
(`route_handler.rs:182`; only `new_coordinator` sets it). The registry-miss relay
path in `io.rs:186` therefore always fails with "No Iroh endpoint configured for
relay forwarding" on a substrate. A1's cross-node hop needs an outbound endpoint,
so §6 fixes this — pre-existing bug, fixed as a dependency, called out in
`status.md`.

**F8 — perf budget "Universal Proxy call (JSON-RPC, same-node) < 5 ms p99
round-trip incl. WIT⇄JSON both ways" is ambiguous** (in-process dispatch, or a
loopback QUIC round-trip to oneself?). **Recommendation:** interpret as
**in-process local dispatch** (`ProxyRouter::invoke` → registry hit → native/WASM
dispatch → `Value`), which is what a same-node call actually does; benchmark that
and say so in `status.md`.

**F9 — stale line anchors** in `task.md`'s Current State Inventory
(`rpc/src/native.rs:15-18`, `synsvc_native.rs:588`, `io.rs:61,90-92`,
`dispatch.rs:122-123`) all moved with B0. Cosmetic; refresh when updating the
doc.

**F10 — a cross-node test does *not* need Playwright.** An earlier draft of this
plan assumed the mainline-DHT port-`6881` collision (documented in
`substrate/tests/common/mod.rs`) made two in-process substrates impossible, and
routed the cross-node proxy test through the 4-node Playwright multi-hop
harness. Wrong: `crates/coordinator_iroh/tests/multi_hop_relay.rs` already runs
two full substrate nodes plus coordinators in one process with
`enable_bep0044_dht = false`. §11.6 uses that instead — cheaper, and it runs
inside `cargo test --workspace`.

**F11 — the guest capability gate must not foreclose M04B B3.** A gate keyed on
"caller service == target service" applied to *all* proxy calls would refuse the
cross-service `data-layer` fetch that M04B's B3 relationship-proof lookup **is**
— while this very slice claims to co-design its request shape for B3. The gate
is therefore scoped to `CallOrigin::Guest` (§5.3), with a `CallOrigin::Native`
escape hatch for substrate-internal callers, a `TODO(M04B/FDAE)` at the gate, and
a unit test pinning the B3-shaped case as *allowed* so a later tightening can't
silently re-break it.

## 2. Crate topology (why the trait lives where it does)

```
syneroym-ucan  ←  syneroym-rpc  ←  syneroym-sandbox-wasm  ←  syneroym-router  ←  syneroym-substrate
                       ▲                    ▲                       │
                       │                    │                       │ impl ServiceProxy
                  ServiceProxy         Weak<dyn ServiceProxy>       │  (ProxyRouter)
                  (trait + types)      (held by AppSandboxEngine)   ┘
```

- `router` already depends on `sandbox_wasm`, so the trait **cannot** live in
  `router` (the guest host function needs it). It goes in `syneroym-rpc`,
  which both already depend on, next to `CallerContext`/`NativeInvocation`.
- `AppSandboxEngine` holds `OnceLock<Weak<dyn ServiceProxy>>` — **`Weak`, set at
  the composition root**, exactly like the existing `self_weak` /
  `MessagingContext::engine` pattern. A strong `Arc` here would form
  `RouteHandlerInner → ProxyRouter → AppSandboxEngine → ProxyRouter`, i.e. the
  same uncollectable-cycle class that hung graceful shutdown in Slice 6B.
- `ProxyRouter` holds `Weak<AppSandboxEngine>` and `WeakNativeDispatchRegistry`
  for the same reason.

## 3. `syneroym-rpc`: the proxy contract (new `crates/rpc/src/proxy.rs`)

```rust
//! Transport-agnostic service-proxy contract (M04A Slice A1).

use std::{fmt::Debug, time::Duration};
use serde_json::Value;
use crate::CallerContext;

/// Reserved wire tag (A.5/A.7): only `JsonRpcV1` exists in M4. A future wRPC
/// wire adds a variant here plus a `RemoteHop` impl — no other type changes.
#[derive(Clone, Copy, Debug, Default, PartialEq, Eq)]
pub enum ProxyProtocol {
    #[default]
    JsonRpcV1,
}

impl ProxyProtocol {
    pub const JSON_RPC_V1_TAG: &'static str = "json-rpc/v1";

    /// `None`/`"json-rpc/v1"` → `JsonRpcV1`; anything else → `Err(tag)`, which
    /// the caller turns into `ProxyError::UnsupportedProtocol`.
    pub fn parse(tag: Option<&str>) -> Result<Self, String> { … }
}

/// Who originated a proxy call. **Host-set, never guest-settable** — the guest
/// host function (§9) always constructs `Guest`, so a component cannot claim
/// `Native` to slip past the §5.3 capability gate.
#[derive(Clone, Debug, PartialEq, Eq)]
pub enum CallOrigin {
    /// A WASM component calling out through `syneroym:proxy`. `service_id` is
    /// the guest's own `component_id` — the **raw** service id, not the
    /// `"system:"`-prefixed synthetic DID that `CallerContext::service_system`
    /// puts in `caller_did`.
    Guest { service_id: String },
    /// Substrate-internal: the FDAE policy engine's relationship-proof fetch
    /// (M04B B3), control-plane internals, tests. Not subject to the §5.3
    /// guest gate — enforcement for these lives at the data-owning node
    /// (ADR-0016 §6; M04B "enforce at the data-owning node").
    Native,
}

/// A cross-service call. Locally constructed; `caller` is **never**
/// wire-serialized (ADR-0016 §6) — only `caller.proof` (signed material)
/// crosses a hop, and the destination re-verifies it.
#[derive(Clone, Debug)]
pub struct ProxyRequest {
    pub target_service: String,
    pub interface: String,
    pub method: String,
    pub params: Value,
    pub caller: CallerContext,
    pub origin: CallOrigin,
    pub protocol: ProxyProtocol,
    /// Retry eligibility. Transport failures are retried with backoff only
    /// when `true`; a callee-returned error is never retried. Failed-after-
    /// retries fails directly — no queueing (DLQ is M5).
    pub idempotent: bool,
    /// Per-call deadline. `None` → `DEFAULT_PROXY_CALL_TIMEOUT`. M04B's B3
    /// policy watchdog sets this explicitly (fail-closed on timeout).
    pub timeout: Option<Duration>,
}

#[derive(Debug, thiserror::Error)]
pub enum ProxyError {
    #[error("unknown service: {0}")]
    ServiceNotFound(String),
    #[error("unsupported protocol '{0}' (this node speaks json-rpc/v1)")]
    UnsupportedProtocol(String),
    #[error("target endpoint kind is not callable over the proxy: {0}")]
    UnsupportedTarget(String),
    #[error("permission denied: {0}")]
    PermissionDenied(String),
    #[error("transport error: {0}")]
    Transport(String),
    /// The callee answered with a JSON-RPC error — definitive, never retried.
    #[error("callee error {code}: {message}")]
    Callee { code: i32, message: String, data: Option<Value> },
    #[error("timed out after {0:?}")]
    Timeout(Duration),
    #[error("internal proxy error: {0}")]
    Internal(String),
}

impl ProxyError {
    /// JSON-RPC code for surfacing this over a wire/HTTP boundary.
    pub fn code(&self) -> i32 {
        match self {
            Self::ServiceNotFound(_)      => -32601,
            Self::UnsupportedProtocol(_)  => UNSUPPORTED_PROTOCOL_RPC_CODE, // -32091
            Self::UnsupportedTarget(_)    => -32093,
            Self::PermissionDenied(_)     => -32010, // same shape as data-layer denial
            Self::Unauthenticated_placeholder => unreachable!(), // (not a variant; see §7)
            Self::Transport(_) | Self::Timeout(_) => PROXY_TRANSPORT_RPC_CODE, // -32092
            Self::Callee { code, .. }     => *code,
            Self::Internal(_)             => -32603,
        }
    }
}

pub const UNSUPPORTED_PROTOCOL_RPC_CODE: i32 = -32091;
pub const PROXY_TRANSPORT_RPC_CODE: i32 = -32092;
pub const UNSUPPORTED_TARGET_RPC_CODE: i32 = -32093;
pub const DEFAULT_PROXY_CALL_TIMEOUT: Duration = Duration::from_secs(30);

#[async_trait::async_trait]
pub trait ServiceProxy: Send + Sync + Debug {
    /// Returns the callee's JSON-RPC `result` value on success.
    async fn invoke(&self, request: ProxyRequest) -> Result<Value, ProxyError>;
}
```

(The `Unauthenticated_placeholder` line above is illustrative noise — delete it;

### 3.1 `CallerContext` gains forwardable proof (`crates/rpc/src/native.rs`)

ADR-0016 §6 / B0 plan §9.5: capabilities never cross the wire; **signed proofs
do**, and are re-verified at the destination. B0's `CallerContext` kept no proof
material, so A1 adds exactly that — the bytes the inbound preamble already
carried, kept opaque so `syneroym-rpc` does not need a `syneroym-identity`
dependency:

```rust
/// Signed, forwardable proof of the caller's identity — verbatim what the
/// inbound preamble carried (ADR-0016 §6). A cross-node proxy hop re-presents
/// it on the outbound preamble; the destination re-verifies it with
/// `HandshakeVerifier::verify_preamble` and builds a **fresh** `CallerContext`.
/// `None` for substrate-injected callers (`local_elevated`/`service_system`),
/// which therefore cannot be impersonated across a hop.
///
/// TODO(B1 — UCAN chains): ADR-0016 §6 says a hop carries "the caller's DID
/// **and its UCAN token(s) / delegation proof**". A1 can only carry the
/// delegation half, because `CapabilityToken` does not exist yet (B0 shipped
/// `Capability`/`SessionContext` only; `issue`/`verify_chain` are B1). When B1
/// lands, this struct gains the token chain and **the outbound preamble in
/// `ProxyRouter::invoke_remote` (§5.5) must carry it too** — today's preamble
/// has no slot for it, so B1 must decide between a new `ucan=` query param and
/// a JSON-RPC envelope field. Do not let B1 rediscover this from scratch: the
/// three places that change together are this struct, `build_caller`
/// (`router/src/route_handler/io.rs`), and `invoke_remote`'s preamble builder.
#[derive(Clone, Debug, PartialEq, Eq)]
pub struct CallerProof {
    /// Hex-encoded ed25519 public key (the preamble's `pubkey=`).
    pub pubkey_hex: String,
    /// JSON `DelegationCertificate` (the preamble's `delegation=`, pre-hex).
    pub delegation_json: Option<String>,
}

pub struct CallerContext {
    pub caller_did: String,
    pub app_instance: Option<String>,
    pub session: SessionContext,
    pub auth: AuthLevel,
    pub proof: Option<CallerProof>,   // NEW (A1)
}
```

`CallerContext`" — that sentence's intent was "A1 must not put `CallerContext`
(capabilities included) on the wire." `proof` is the mechanism §9.5 itself

### 4.3 Inbound route switches to typed results

`route_handler/dispatch.rs:109-137`, the `(JsonRpcToWasm, WasmComponent)` arm:
`execute_wasm(...)` → `execute_wasm_json(...)`, and
`result: Value::String(wasm_result)` → `result: wasm_result`.

### 5.2 `ServiceProxy::invoke` — top level

```
async fn invoke(&self, req: ProxyRequest) -> Result<Value, ProxyError> {
    // 1. Protocol gate (the minimal [LFC-VER] behavior; A.7).
    if req.protocol != ProxyProtocol::JsonRpcV1 { return Err(UnsupportedProtocol(tag)) }

    // 2. Capability gate: a WASM guest must not reach another service's
    //    native capabilities through the proxy (§5.3).
    self.check_native_capability_gate(&req)?;

    metrics::counter!("substrate.proxy.calls").increment(1);
    let started = Instant::now();

    // 3. Local first: the endpoint registry is authoritative for services
    //    hosted on this node (this is also the <5ms same-node path).
    let outcome = match self.registry.lookup(&req.target_service, &req.interface) {
        Some((endpoint, canonical_iface)) => self.invoke_local(&req, endpoint, canonical_iface).await,
        None => self.invoke_remote(&req).await,     // §5.4
    };

    metrics::histogram!("substrate.proxy.duration_ms").record(started.elapsed()…);
    if outcome.is_err() { metrics::counter!("substrate.proxy.errors").increment(1); }
    outcome
}
```

### 5.3 The guest native-capability gate (security-critical, do not omit)

`SynSvcNativeService` is registered **per deployed service** in
`native_dispatch` keyed by that service's id, and its `data-layer` arm opens
*that service's* SQLite DB. Without a gate, guest `A` could call
`proxy.call(service = B, interface = "data-layer", method = "query")` and — under
B0's "any verified identity passes" rule — read B's database. A1 must not open
that hole.

The gate applies to **`CallOrigin::Guest` only**. A substrate-internal
(`CallOrigin::Native`) call to another service's `data-layer` is exactly what
M04B's Slice B3 relationship-proof fetch is, so gating it here would foreclose
the scenario A1 is explicitly supposed to co-design for (co-design seam #2).
Native-origin calls are authorized at the **data-owning node** — the destination
re-verifies the forwarded proof (§5.5) and, once M04B lands, runs the FDAE
policy inside the callee's own `data-layer` dispatch. That is the correct
enforcement point; a coarse origin check in the proxy would be redundant there
and wrong for B3.

```rust
/// `data-layer`, `vault`, `app-config`, `blob-store`, `messaging`,
/// `http-native` — the reserved names every deployed service auto-registers
/// (moved to `syneroym_core`; see the note below).
use syneroym_core::local_registry::NATIVE_CAPABILITY_INTERFACES;

/// TODO(M04B/FDAE): this is an interim, coarse, fail-closed gate — "a guest may
/// only reach its **own** service's native capabilities through the proxy."
/// M04B replaces it with real per-caller/per-row policy evaluated against
/// `caller.session` at the data-owning node, at which point a guest-originated
/// cross-service `data-layer` read becomes expressible (and filtered), not
/// refused outright. Do not widen this gate before that policy exists.
fn check_native_capability_gate(&self, req: &ProxyRequest) -> Result<(), ProxyError> {
    // Substrate-internal callers (M04B B3's proof fetch, control-plane
    // internals) are authorized at the destination, not here.
    let CallOrigin::Guest { service_id } = &req.origin else { return Ok(()) };

    if !NATIVE_CAPABILITY_INTERFACES.contains(&req.interface.as_str()) { return Ok(()) }

    // Compare the guest's **raw** component_id against the target. NOT
    // `caller.caller_did`: `CallerContext::service_system` sets that to
    // `"system:<service_id>"` (`rpc/src/native.rs:84-95`), which can never
    // equal a plain service id — using it would reject a component's calls to
    // its own service too.
    if service_id == &req.target_service { return Ok(()) }

    Err(ProxyError::PermissionDenied(format!(
        "component '{service_id}' may not reach native capability '{}' on service '{}' through \
         the proxy (cross-service native-capability policy is FDAE/M04B)",
        req.interface, req.target_service)))
}
```

(A component reaching *its own* native capabilities through the proxy is
permitted but pointless — it has direct host imports for them. The arm exists so
the gate's rule is "not yours ⇒ denied" rather than a blanket "no native
capabilities from guests", which would be a different, harder-to-relax rule.)

`NATIVE_CAPABILITY_INTERFACES` currently lives in
`crates/control_plane/src/service/orchestration.rs:47` (private). `router` must
not depend on `control_plane` (that would invert the dep graph). **Move the const
to `syneroym-core`** (e.g. `syneroym_core::local_registry::NATIVE_CAPABILITY_INTERFACES`)
and re-export/import it from both `control_plane` and `router`, so the two lists
cannot drift. (`router/tests/native_dispatch_identity.rs:169` has a third copy —
point it at the shared const too.)

**Note on the local WASM path and caller identity:** `execute_wasm_json` →

callees, and is "the callee acts as itself" for WASM callees.**

### 5.5 Remote dispatch + the `RemoteHop` seam

/// response". The transport-agnostic seam the task asks for: a future wRPC wire
/// (A.5) is a second impl + a second `ProxyProtocol` variant, nothing else.

pub trait RemoteHop: Send + Sync + Debug {

        addr: &EndpointAddr,
        preamble: &RoutePreamble,
        request: &JsonRpcRequest,
        timeout: Duration,

}

#[derive(Debug)]
pub struct IrohHop { endpoint: Endpoint, retry_policy: RetryPolicy }

`IrohHop::call` (mirrors `SyneroymClient::request_raw`, which `router` cannot use
— `sdk` depends on `router`, not the reverse):

```
let conn = net_iroh::connect_with_retry(&self.endpoint, addr.clone(), SYNEROYM_ALPN, &self.retry_policy)
              .await.map_err(|e| ProxyError::Transport(e.to_string()))?;
let (mut send, mut recv) = conn.open_bi().await.map_err(Transport)?;
send.write_all(preamble.to_preamble_line().as_bytes()).await.map_err(Transport)?;
framing::write_frame(&mut send, &serde_json::to_vec(request)?).await.map_err(Transport)?;
send.finish().map_err(Transport)?;
let frame = time::timeout(timeout, framing::read_frame(&mut recv)).await
              .map_err(|_| ProxyError::Timeout(timeout))?.map_err(Transport)?;
if frame.is_empty() { return Err(ProxyError::Transport("empty response frame".into())) }
// Success or error envelope — a JSON-RPC error is a *definitive* answer.
if let Ok(ok) = serde_json::from_slice::<JsonRpcResponse>(&frame) { return Ok(ok.result) }
let err: JsonRpcErrorResponse = serde_json::from_slice(&frame)
    .map_err(|e| ProxyError::Transport(format!("malformed response: {e}")))?;
Err(ProxyError::Callee { code: err.error.code, message: err.error.message, data: err.error.data })
```

`ProxyRouter::invoke_remote`:

```
// 1. Resolve the target's Iroh address via the community registry / DHT.
//    Factor the existing block in io.rs:162-180 (registry lookup -> pick the
//    Iroh mechanism -> apply relay URL) into a shared
//    `net_iroh::resolve_iroh_addr(&RegistryClient, service_id) -> Result<EndpointAddr>`
//    and call it from both places.
let addr = net_iroh::resolve_iroh_addr(&self.registry_client, &req.target_service)
    .await.map_err(|_| ProxyError::ServiceNotFound(req.target_service.clone()))?;

// 2. Build the outbound preamble. Identity: forward the caller's *signed
//    proof* verbatim when it has one (ADR-0016 §6 — the destination
//    re-verifies with verify_preamble and builds a fresh CallerContext);
//    otherwise present this node's own identity. Capabilities never cross.
let mut preamble = RoutePreamble::binary_json_rpc(&req.target_service, &req.interface);
match &req.caller.proof {
    Some(p) => {
        preamble.pubkey = Some(p.pubkey_hex.clone());
        preamble.delegation = p.delegation_json.as_deref()
            .and_then(|j| DelegationCertificate::from_json(j).ok());
    }
    None => preamble.pubkey = Some(hex::encode(self.node_identity.public_key().to_bytes())),
}

// 3. Retry loop. Only *transport* failures are retried, and only when the
//    caller declared the call idempotent. Failed-after-retries fails directly
//    (no DLQ — M5).
let attempts = if req.idempotent { self.retry_policy.max_attempts.max(1) } else { 1 };
let mut backoff = self.retry_policy.initial_backoff_ms;
for attempt in 1..=attempts {
    match self.hop.call(&addr, &preamble, &json_rpc_request, timeout).await {
        Ok(v) => return Ok(v),
        Err(e) if !matches!(e, ProxyError::Transport(_) | ProxyError::Timeout(_)) => return Err(e),
        Err(e) if attempt == attempts => return Err(e),
        Err(e) => {
            warn!(attempt, max = attempts, error = %e, "proxy call failed; retrying");
            metrics::counter!("substrate.proxy.retries").increment(1);
            time::sleep(Duration::from_millis(retry::calculate_jittered_backoff(backoff))).await;
            backoff = ((backoff as f64 * self.retry_policy.backoff_multiplier) as u64)
                       .min(self.retry_policy.max_backoff_ms);
        }
    }
}
```

Note the double-retry interaction: `connect_with_retry` already retries
*connection establishment* with the same policy; the loop above retries the whole
*call*. That is intentional (connect vs. request-level failures) but must be
called out in `status.md` — worst-case attempts is `max_attempts²` for an
unreachable peer. Consider passing a single-attempt policy to `IrohHop`'s
`connect_with_retry` so the outer loop owns backoff exclusively. **Recommended:
do that** (`RetryPolicy { max_attempts: 1, ..policy }` inside `IrohHop`), and say
so in the doc comment.

No connection cache in A1 (one `open_bi` per call, per-call connect). The
`<5 ms p99` budget is same-node (F8); the remote path's cost is measured and
reported, and a `DashMap<service_id, Connection>` cache is noted as the obvious
next hook point.

```wit
package syneroym:proxy@0.1.0;

/// The Universal Proxy (M04A Slice A1). A component calls another service —
/// local or on another node — without knowing where it lives. `params` and the
/// success value are JSON text; the **callee** binds them against its real WIT
/// signature (records/variants/lists/options — see `conversions.rs`), so the
/// call is typed at the dispatch boundary even though the wire is JSON-RPC.
/// A binary wRPC wire is deferred (M04A Decision Register A.5); when it lands,
/// `call-options.protocol` selects it and this interface does not change.
interface proxy {
    record callee-error {
        code: s32,
        message: string,
        data: option<string>,
    }

    variant proxy-error {
        service-not-found(string),
        unsupported-protocol(string),
        unsupported-target(string),
        permission-denied(string),
        transport(string),
        timed-out,
        callee(callee-error),
        internal(string),
    }

    record call-options {
        /// Reserved wire tag. `none` or "json-rpc/v1" today; any other value
        /// returns `unsupported-protocol` (negotiation is deferred — A.7).
        protocol: option<string>,
        /// Retry transport failures with exponential backoff. Only set this
        /// for calls that are safe to repeat; a callee-returned error is never
        /// retried. Defaults to false.
        idempotent: bool,
        /// Per-call deadline; defaults to the host's 30s budget.
        timeout-ms: option<u32>,
    }

    /// `service` is the target's DID (or a registry alias). `interface` is a
    /// WIT interface name the target registered at deploy time. Native
    /// capabilities (`data-layer`, `vault`, …) of *another* service are
    /// refused with `permission-denied` — a component reaches its own through
    /// its host imports.
    call: func(
        service: string,
        interface: string,
        method: string,
        params: string,
        options: option<call-options>,
    ) -> result<string, proxy-error>;
}
```

```rust
impl proxy::Host for HostState {
    async fn call(
        &mut self,
        service: String,
        interface: String,
        method: String,
        params: String,
        options: Option<proxy::CallOptions>,
    ) -> Result<String, proxy::ProxyError> {
        let proxy = self.service_proxy.upgrade()
            .ok_or_else(|| proxy::ProxyError::Internal("proxy unavailable".into()))?;

        let params: Value = if params.trim().is_empty() { Value::Null }
                            else { serde_json::from_str(&params)
                                     .map_err(|e| proxy::ProxyError::Internal(format!("params must be JSON: {e}")))? };
        let opts = options.unwrap_or_default();
        let protocol = ProxyProtocol::parse(opts.protocol.as_deref())
            .map_err(proxy::ProxyError::UnsupportedProtocol)?;

        let req = ProxyRequest {
            target_service: service,
            interface,
            method,
            params,
            // The component acts as itself. It does NOT inherit the identity of
            // whoever invoked it (no U→X delegation exists in B0's model), so a
            // proxied call cannot be used to escalate to the original caller's
            // rights. Real caller-delegation is B1/UCAN.
            caller: CallerContext::service_system(&self.component_id),
            // Always `Guest` — this is the only construction site a component
            // can reach, so the §5.3 gate cannot be bypassed from guest code.
            origin: CallOrigin::Guest { service_id: self.component_id.clone() },
            protocol,
            idempotent: opts.idempotent,
            timeout: opts.timeout_ms.map(|ms| Duration::from_millis(ms.into())),
        };

        let value = proxy.invoke(req).await.map_err(map_proxy_error)?;   // ProxyError -> WIT variant
        Ok(match value { Value::String(s) => s, other => other.to_string() })
    }
}
```

### 11.3 Unit — `crates/router/src/proxy.rs` (with a `MockHop`)
- local native target: the callee `NativeService` double receives
  `invocation.caller.caller_did` == the request's caller, and the declared
  `interface`/`method`/`params`.
- local WASM target: skip-if-artifact-missing, `greeter`-style component returns
  a typed value.
- unknown service (registry miss + registry-client failure) → `ServiceNotFound`,
  and `MockHop` was **never** called.
- `protocol: "wrpc"` → `UnsupportedProtocol`, `MockHop` never called.
- **native-capability gate** (§5.3), four cases against a recording
  `NativeService` double, each asserting the double was/wasn't invoked — not
  just the returned `Result`:
  - `origin: Guest { service_id: "svc-a" }`, target `("svc-b", "data-layer")`
    → `PermissionDenied`, callee **never dispatched**.
  - `origin: Guest { service_id: "svc-a" }`, target `("svc-a", "data-layer")`
    → allowed, callee dispatched. (This is the case that would have failed
    against a `caller_did`-based comparison — `service_system` puts
    `"system:svc-a"` there — so it is a regression test for that bug, not just
    a positive control.)
  - `origin: Guest { .. }`, target `("svc-b", "some-app-interface")` (not a
    reserved native capability) → allowed.
  - **`origin: Native`, target `("svc-b", "data-layer")` → allowed** — the
    M04B B3 relationship-proof-fetch shape. This test is the guard that stops a
    future tightening of the gate from silently re-breaking B3.
- retry: `MockHop` failing with `Transport` — `idempotent: true` ⇒
  `max_attempts` calls; `idempotent: false` ⇒ exactly 1; `Callee` error ⇒ exactly
  1 even when idempotent.
- proof forwarding: a caller with `proof: Some(..)` produces an outbound preamble
  carrying that exact `pubkey`/`delegation` (assert on the preamble `MockHop`
  received); a caller with `proof: None` produces the **node's own** pubkey.

### 11.6 Cross-node proxy call (reference scenario step 20) — **in-process, not Playwright**

`crates/coordinator_iroh/tests/multi_hop_relay.rs` already stands up two full
substrate nodes (`Sz`, `Sx`) plus a registry and two coordinators **in one
process**, driving them through `RouteHandler::init` directly. It avoids the
mainline-DHT port-`6881` collision that forces `substrate/tests/common/mod.rs`
to serialize (`config.substrate.enable_bep0044_dht = false` on every node), so
the constraint I originally assumed — "an in-process two-substrate test is
impossible" — does not hold. This is much cheaper than a Playwright spec and
runs in `cargo test --workspace`.

Add `test_cross_node_proxy_call` to that file (or a sibling
`crates/coordinator_iroh/tests/universal_proxy.rs` reusing its harness helpers):
deploy a WASM component on `Sx` that calls `proxy.call(service = did_z,
interface = <greeter iface>, method = "greet", params = …)`, and assert the
typed result comes back from `Sz`'s component. Assert `Sz` saw a **verified,
non-anonymous** caller (the forwarded proof / `Sx`'s node identity, per §5.5) —
i.e. B0's mandatory-verify gate is satisfied by a proxied hop, which is the one
thing that could plausibly break.

Because `Sz`/`Sx` reach each other through the coordinator relay in that harness,
this also exercises the §6 endpoint fix end to end.

**Playwright (`mise run test:e2e`) is then only a regression gate** — run it, but
do not add a new spec unless the in-process test can't reach some part of the
flow (e.g. the gateway hop). Note it either way in `status.md`.
